# Stellar Burgers — Автоматизация тестирования

Этот репозиторий содержит автоматизацию тестирования веб-приложения **Stellar Burgers**. Проект выполнен в рамках дипломной работы по курсу QA.  

---

## Структура проекта

- `develop1` — юнит-тесты для класса **Burger** с использованием **JUnit 4**, **Mockito**, параметризацией и отчётом **JaCoCo**.
- `develop2` — автотесты для **API Stellar Burgers** с использованием **REST Assured**, **JUnit 4** и **Allure**.
- `develop3` — автотесты для **UI Stellar Burgers** с использованием **Selenium**, **Page Object** и **Allure**.

---

## Часть 1. Юнит-тесты

**Цель:** покрыть юнит-тестами класс `Burger` на 100%  
**Используемые инструменты:**
- JUnit 4
- Mockito
- Параметризация тестов
- JaCoCo (отчёт о покрытии)

**Реализовано:**
- Проверка добавления, удаления и перемещения ингредиентов
- Проверка расчёта стоимости бургера
- Проверка формирования чека

---

## Часть 2. Автотесты API

**Цель:** протестировать ключевые эндпоинты API Stellar Burgers  

**Реализовано:**
- **Создание пользователя:**
  - Новый пользователь
  - Пользователь уже зарегистрирован
  - Не заполнены обязательные поля
- **Вход пользователя:**
  - Успешный вход
  - Некорректный логин/пароль
- **Создание заказа:**
  - С авторизацией и без
  - С ингредиентами и без
  - С неверным хешем ингредиентов

**Инструменты:** REST Assured, JUnit 4, Allure  

---

## Часть 3. Автотесты UI

**Цель:** протестировать функциональность веб-приложения Stellar Burgers  

**Реализовано:**
- **Регистрация пользователя**
  - Успешная регистрация
  - Ошибка при некорректном пароле (<6 символов)
- **Вход в аккаунт**
  - Через кнопку "Войти в аккаунт"
  - Через кнопку "Личный кабинет"
  - Через кнопки в форме регистрации и восстановления пароля
- **Раздел "Конструктор"**
  - Переходы между секциями: Булки, Соусы, Начинки

**Инструменты:** Selenium WebDriver, Page Object, Allure
